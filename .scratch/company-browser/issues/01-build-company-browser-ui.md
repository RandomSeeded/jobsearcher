# Build Company Browser UI

> *This was generated by AI during triage.*

## Summary

Build the company browser — a React + TypeScript + Vite app in `client/` that lets the user browse, filter, and vote on companies from `data/contenders/*.yaml`. All design and architecture decisions have been resolved via a grilling session.

---

## Agent Brief

### Goal

Implement the company browser UI (Variant M) with a thin local Express server backing it.

### Stack

- **Client:** React + TypeScript + Vite, in `client/`
- **Server:** Express, thin CRUD wrapper over `data/contenders/*.yaml`
- **Routing:** React Router — `/` (browser), `/triage` (stub only)

### Layout — Variant M

Two-column layout:

**Left — main area:**
- Company tile grid (`auto-fill` columns)
- Grouped: "Love & Like" section first, then "Others"
- Tile shows: name, vote emoji, AI category badge, company quality stars
- Click tile → detail drawer slides in from right (420px wide, overlays main area)

**Right — insights sidebar (always visible, 280px):**
- "At a glance" stats: total tracked, love+like count, in-pipeline count
- "You favor" — [[Preference Profile]] tags derived from vote history (AI category, location)
- "Start triage" button → navigates to `/triage`

**Drawer (company detail):**
- Name, URL, location, employees, company quality, stage, AI category
- Full notes
- Vote buttons (love / like / neutral / not_sure_yet / dislike) — persisted via server
- Close button

**Search:** ⌘K opens command palette overlay — fuzzy match by name and AI category. Clicking result opens drawer.

### Server endpoints

```
GET  /api/companies           → all companies parsed from data/contenders/*.yaml
PATCH /api/companies/:name    → update a single field (vote, stage, etc.) in its YAML file
```

### Data shape (from YAML)

Key fields: `company`, `vote`, `stage`, `location`, `employees`, `link`, `ai_category`, `company_quality`, `last_outreach`, `notes`

Vote values: `love | like | neutral | not_sure_yet | dislike` or absent (NYR)

### Preference Profile (insights sidebar)

Compute at runtime from love/like companies:
- `catScore`: count of AI categories, weighted love=2, like=1
- `locScore`: location frequency
- `sizeGroups`: employee count bucketed (`<30`, `30-80`, `80+`)

No persistence yet — derived fresh each render.

### Triage flow (`/triage`)

Stub route only. Render a placeholder: "Decision Queue — coming soon." Add a "Back" link to `/`.

### Out of scope

- Stage updates via the UI (Notion handles this)
- Discover more / LLM discovery (future)
- Preference Profile persistence (planned, not now)
- Anything beyond companies CRUD on the server

### Definition of done

- `client/` scaffolded with Vite + React + TypeScript
- Server runs alongside (`npm run server` or similar), proxied through Vite dev server
- All real companies from `data/contenders/*.yaml` render in the browser
- Voting in the drawer persists back to YAML
- ⌘K palette searches by name and AI category
- `/triage` renders a stub
- `npm run dev` starts both client and server
